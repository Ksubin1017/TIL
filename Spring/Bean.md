# 스프링 컨테이너
애플리케이션 설정 정보를 바탕으로 객체(빈)를 대신 생성 및 관리 의존성을 연결 생명주기를 책임지는 스프링의 핵심 관리 공간
스프링 컨테이너는 개발자가 직접 객체를 생성하고 관리하는 부담을 줄여준다.

스프링 컨테이너는 BeanFactory와 ApplicationContext로 구분되는데, ApplicationContext는 BeanFactory의 확장된 형태로
국제화 지원, 이벤트 발행 등 추가적인 기능을 제공한다.

# 스프링 빈
스프링 컨테이너에 의해 관리되는 재사용 가능한 컴포넌트
스프링 간 객체가 의존 관계를 관리하도록 하기 위해 사용하며 객체가 의존 관계를 등록할 때 스프링 컨테이너에섯 해당하는 빈을 찾아 그 빈과 의존성을 생성한다.

기본적으로 싱글톤 스코프를 가진다. 동일한 빈 정의에 대해 스프링 컨테이너는 단일 인스턴스만을 생성하고 이것을 애플리케이션 내에서 공유한다.

XML, Java Config, 어노테이션 등 다양한 방법으로 설정할 수 있다.

## 생명주기
### 생성
스프링 컨테이너가 설정 정보를 바탕으로 빈을 생성
컨테이너 초기화 시점에 빈 객체가 생성되고 다른 빈과 의존 관계가 설정되지 않은 상태이다.

### 의존 관계 주입
빈 객체가 생성된 후 스프링 컨테이너는 빈 객체 간 의존 관계를 설정한다.
@Autowired, 생성자 주입 등을 통해 이루어지며 객체가 사용할 준비를 하기 시작한다.

### 사용
실제 서비스 로직에서 빈이 사용되는 단계

### 소멸
스프링 컨테이너가 종료되면서 빈 객체도 소멸되는 단계
빈 객체의 자원 해제 작업이 이루어진다.

## 스프링 컨테이너와 빈 객체의 생명주기 관계
컨테이너가 초기화될 때 빈 객체들이 생성되고 컨테이너가 종료될 때 빈 객체들도 소멸된다.

### 컨테이너 초기화
빈 객체 생성, 의존 관계 주입, 빈 객체 초기화가 수행된다.

### 컨테이너 사용
애플리케이션이 실행되는 동안 빈 객체들이 사용된다.

### 컨테이너 종료
빈 객체가 소멸된다.
