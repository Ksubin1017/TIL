## DNS
IP 주소만으로는 호스트의 특징을 나타내기 어렵고 IP 주소는 언제든 바뀔 수 있다.  
따라서 문자열 형태의 호스트를 특정할 수 있는 정보인 도메인 네임을 사용한다.

도메인 네임과 그에 대응하는 IP 주소는 네임 서버에서 관리되고 도메인 네임을 관리하는 네임 서버를 DNS라고 한다.

### 도메인 네임의 계층 구조
`www.example.com.`

하나의 도메인 네임은 점(.)을 기준으로 계층적으로 분류되어 있다.
- 루트 도메인: 도메인 네임의 마지막 점(.)
- 최상위 도메인: com
- 2단계 도메인: example
- 3단계 도메인: www

## 자원과 URI/URL

### 자원
네트워크 상의 메시지를 통해 주고받는 최종 대상  
-> 두 호스트가 네트워크를 통해 서로 정보를 주고받을 때 송수신 하는 대상

### URI
웹 상에서 자원을 식별하기 위한 정보

- URN: 이름을 기반으로 자원을 식별하는 URI
- URL: 위치를 기반으로 자원을 식별하는 URI

예시 URL  
`http://www.example.com:1017/over/there?name=KIM#section`

### scheme
자원에 접근하는 방법을 나타내는 부분  
예시에서는 HTTP를 사용하여 자원에 접근함을 나타낸다.

### authority
호스트를 특정할 수 있는 IP 주소 혹은 도메인 네임이 명시된다.

### path
자원이 위치하고 있는 경로가 명시된다.  
슬래시(/)를 기준으로 계층적으로 표현되며 최상위 경로 또한 슬래시로 표현된다.

### query
URL에 대한 매개변수 역할을 하는 문자열  
scheme, authority, path만으로 표현하기 어려운 추가 정보  
`?`로 시작하는 `<키=값>` 형태의 데이터로 앰퍼샌드(&)를 사용하여 여러 쿼리 문자열을 연결할 수 있다.

### fragment
자원의 일부분을 가리키기 위한 정보
