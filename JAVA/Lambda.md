# 람다식

함수형 프로그래밍: 함수를 정의하고 함수를 데이터 처리부로 보내 데이터를 처리하는 기법
데이터 처리부는 데이터만 가지고 있고 처리 방법이 정해져 있지 않아 외부에서 제공된 함수에 의존한다.

동일한 데이터라도 함수 A, 함수 B의 결과는 다를 수 있고 이것은 데이터 처리의 다형성으로 볼 수 있다.

```java
람다식: (매개변수, ...) -> {처리 내용}
```
람다식은 위 코드와 같이 데이터 처리부에 제공되는 함수 역할을 하는 매개변수를 가진 중괄호 블록으로,
데이터 처리부는 람다식을 받아 매개변수에 데이터를 대입하고 중괄호를 실행시켜 처리한다.

```java
public interface Calculable {
  // 추상 메서드
  void calculate(int x, int y);
}

// Calculable 인터페이스의 익명 구현 객체
new Calculable() {
  @Override
  public void calculate(int x, int y) { 처리 내용 }
};

// 람다식 변환
(x, y) -> { 처리 내용 };
```

인터페이스의 익명 구현 객체를 람다식으로 표현하려면 인터페이스가 단 하나의 추상 메서드만 가져야한다.
인터페이스가 단 하나의 추상 메서드를 가질 때, 함수형 인터페이스라 하고 이를 보장하기 위해서는 @FunctionalInterface 어노테이션을 붙이면 된다.

## 매개변수가 없는 람다식
```java
() -> {
    실행문;
    실행문;
}

() -> 실행문
```
함수형 인터페이스의 추상 메서드에 매개변수가 없을 경우 위 코드와 같이 작성할 수 있으며 실행문이 두 개 이상일 경우 중괄호를 생략할 수 없다.

## 매개변수가 있는 람다식

```java
(매개변수, ...) -> {
    실행문;
    실행문;
}

(매개변수, ...) -> 실행문
매개변수 -> 실행문
```
매개변수를 선언할 때 타입은 생략할 수 있고 매개변수가 하나일 때는 괄호를 생략할 수 있으며 이때는 타입을 붙일 수 없다.

리턴값이 있는 람다식
```java
(매개변수, ...) -> {
    실행문;
    return 값;
}

(매개변수, ...) -> 값;
```
return문 하나만 있을 경우 중괄호와 함께 return 키워드를 생략할 수 있고 리턴값은 연산식 또는 리턴값이 있는 메서드 호출로 대체할 수 있다.
