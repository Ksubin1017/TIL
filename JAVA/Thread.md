## 메인 스레드
모든 자바 프로그램은 메인 스레드가 main() 메서드를 실행하면서 시작된다.
메인 스레드는 main() 메서드의 첫 코드부터 순차적으로 실행하고 main() 메서드의 마지막 코드를 실행하거나 return문을 만나면 실행을 종료한다.
싱글 스레드에서는 메인 스레드가 종료되면 프로세스도 종료되고, 멀티 스레드에서는 실행 스레드가 하나라도 있다면 프로세스는 종료되지 않는다.
메인 스레드가 작업 스레드보다 먼저 종료되더라도 작업 스레드가 계속 실행 중이라면 프로세스는 종료되지 않는다.

## 작업 스레드 생성 및 실행
Thread 클래스

```java
Thread thread = new Thread(Runnable target);
```

Thread 클래스로부터 작업 스레드 객체를 직접 생성하려면 Runnable 구현 객체를 매개값으로 갖는 생성자를 호출하면 된다.
Runnable은 스레드가 작업을 실행할 때 사용하는 인터페이스로 run() 메서드가 정의되어 있으며 구현 클래스는 이 메서드를
재정의해서 스레드가 실행할 코드를 가지고 있어야 한다.

작업 스레드 객체가 생성되었다고 바로 실행되지는 않고, 스레드 객체의 start() 메서드를 호출해야 작업 스레드가 실행된다.

## 실행 양보
스레드가 반복문을 처리할 때, 무의미한 반복을 하는 경우가 있다.
이 경우를 위해 Thread 클래스는 yield() 메서드를 제공한다.
yield()를 호출한 스레드는 대기 상태가 되고 다른 스레드가 실행 상태가 된다.

```java
public void run() {
  while(true) {
    if(work) {
      System.out.println("ThreadA 작업");
    } else {
       Thread.yield();
    }
  }
```

## 스레드 동기화
멀티 스레드의 경우 하나의 객체를 공유해서 작업할 수 있는데 이 경우, 다른 스레드에 의해 객체 내부 데이터가 쉽겨 변경될 수 있다.
스레드가 사용 중인 객체를 다른 스레드가 변경할 수 없도록 하려면 스레드 작업이 끝날 때까지 객체에 잠금을 걸면되는데, 자바에서는 동기화 메서드와 블록을 제공한다.

```java
public synchronized void method() {
    // 하나의 스레드만 실행하는 영역
}
```

## 스레드 제어
두 개의 스레드를 번갈아가며 실행해야할 때 정확한 교대 작업이 필요한 경우 nofify(), wait() 메서드를 사용한다.
notify()는 wait()에 의해 일시 정지된 스레드 중 한 개를 실행 대기 상태로 만들고 notifyAll()은 모든 스레드를 실행 대기 상태로 변경한다.
notify()와 wait()은 동기화 메서드 및 동기화 블록 내에서만 사용 가능하다.
