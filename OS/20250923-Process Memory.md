# 프로세스 (Process)

프로세스는 **실행 중인 프로그램**을 의미한다.  
운영체제가 관리하며, 독립적으로 실행되고 자원을 할당받을 수 있는 단위이다.  
운영체제는 프로세스들에게 적절히 자원을 분배하여 여러 작업을 수행할 수 있도록 한다.

---

## 프로세스의 종류

- **포그라운드 프로세스**  
  사용자가 보는 공간에서 사용자와 상호작용하며 실행

- **백그라운드 프로세스**  
  사용자가 보지 못하는 곳에서 실행  
  - **데몬(Daemon)**  
    백그라운드 프로세스 중 사용자와 상호작용 없이 주어진 작업만 수행하는 특별한 프로세스

---

## 프로세스를 구성하는 메모리

### 1. 커널 영역
- **PCB (Process Control Block)**  
  운영체제가 메모리에 적재된 다수의 프로세스를 관리하기 위해 각 프로세스를 식별할 수 있는 정보를 저장하는 블록

---

### 2. 사용자 영역

- **코드 영역 (텍스트 영역)**  
  - 실행 가능한 명령어가 저장되는 공간  
  - CPU가 읽고 실행할 명령어가 담겨 있어 쓰기가 금지된 읽기 전용(`read-only`)

- **데이터 영역**  
  - 프로그램 실행 동안 유지되는 데이터 저장 공간  
  - 정적 변수, 전역 변수가 주로 저장됨

- **힙 영역**  
  - 개발자가 직접 메모리를 할당받아 사용하는 공간  
  - 실행 도중 자유롭게 할당 및 반환 가능  
  - 반환하지 않으면 **메모리 누수(memory leak)** 발생  
  - 일부 언어는 **가비지 컬렉션(GC)**을 통해 자동으로 해제

- **스택 영역**  
  - 일시적으로 사용할 값들이 저장되는 공간  
  - 매개변수, 지역 변수, 함수 복귀 주소 등이 저장됨  
  - **스택 트레이스(Stack Trace)**  
    특정 시점에 스택에 저장된 함수 호출 정보를 의미
