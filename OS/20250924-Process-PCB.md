# PCB와 프로세스 관리

## PCB (Process Control Block)

운영체제가 메모리에 적재된 다수의 프로세스를 관리하려면 **프로세스를 식별할 수 있는 커널 영역 내의 정보**가 필요하다.  
이 정보가 **프로세스 제어 블록(Process Control Block, PCB)** 이다.

- PCB는 프로세스와 관련된 다양한 정보를 내포하는 구조체의 일종이다.
- 새로운 프로세스가 메모리에 적재(프로세스 생성)될 때 커널 영역에 만들어지고, 프로세스의 실행이 끝나면 폐기된다.

### PCB 내의 정보
- 프로세스 ID (PID)
- 실행 과정에서 사용한 레지스터 값
- 프로세스 상태
- CPU 스케줄링 정보 (우선순위 등)
- 메모리 관련 정보 (프로세스 적재 위치)
- 파일 및 입출력(I/O) 관련 정보

## 문맥 교환 (Context Switching)

- 메모리에 적재된 프로세스들은 한정된 시간 동안 번갈아 가며 실행된다.  
  (= 운영체제에 의해 CPU 자원을 할당받는다.)
- 프로세스의 CPU 사용 시간은 **타이머 인터럽트**에 의해 제한된다.  
  → 타이머 인터럽트란 시간이 끝났음을 알리는 인터럽트이다.

### 문맥(Context)
- 프로세스가 실행 도중 기억해야 하는 정보
- 프로그램 카운터, 각종 레지스터 값, 메모리 정보, 실행 파일, 사용한 I/O 장치 등이 포함된다.
- 이 정보들은 **PCB에 저장**된다.

### 문맥 교환 과정
1. 인터럽트 발생
2. 운영체제가 현재 실행 중인 프로세스의 문맥을 **PCB에 백업**
3. 다음 실행할 프로세스의 문맥을 **PCB에서 복구**
4. CPU 실행을 이어감

이 과정을 **문맥 교환(Context Switching)** 이라고 한다.

## 프로세스의 상태

- **생성 상태 (New)**  
  프로세스를 생성 중인 상태. 메모리에 적재되어 PCB를 할당받은 상태.

- **준비 상태 (Ready)**  
  실행할 수 있지만 아직 차례가 아니어서 대기하는 상태.  
  CPU를 할당받으면 실행 상태가 되며, 준비 → 실행 전환을 **디스패치(Dispatch)** 라고 한다.

- **실행 상태 (Running)**  
  CPU를 할당받아 실행 중인 상태.  
  - 타이머 인터럽트 발생 시 → 다시 준비 상태로 이동  
  - 입출력 요청 시 → 대기 상태로 이동

- **대기 상태 (Blocked)**  
  입출력 작업 요청, 사용 불가능한 자원 요청 등으로 즉시 실행이 불가능한 상태.  
  작업이 완료되면 다시 준비 상태로 돌아가 CPU 할당을 기다린다.

- **종료 상태 (Terminated)**  
  프로세스 실행이 끝난 상태.  
  운영체제가 PCB와 프로세스가 사용한 메모리를 정리한다.

## 블로킹 입출력과 논블로킹 입출력

- **블로킹 입출력**  
  - 입출력 장치에게 작업을 요청한 뒤, 작업이 끝날 때까지 대기 상태로 들어감  
  - 작업 완료 후 준비 상태로 전환되어 실행을 재개

- **논블로킹 입출력**  
  - 입출력 장치에게 작업을 요청한 뒤, 결과를 기다리지 않고 곧바로 이어지는 명령어를 수행
