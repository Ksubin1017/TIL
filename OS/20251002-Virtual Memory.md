# 물리 주소와 논리 주소
CPU와 프로세스는 메모리 하드웨어 상의 물리 주소(실제 주소)가 아닌 논리 주소를 사용한다.

## 논리 주소란?
프로세스마다 부여되는 0번지부터 시작하는 주소 체계. 
- 메모리 상의 모든 프로세스는 0번지부터 시작하는 각각의 논리 주소를 가지고 있다.
- 중복되는 물리 주소의 번지 수는 존재하지 않지만, 논리 주소의 번지 수는 얼마든 중복될 수 있다.
- 중복됨에도 CPU가 해당 프로세스의 메모리를 정확히 찾아갈 수 있게 도와주는 장치가 **메모리 관리 장치(MMU)** 이다.
- MMU는 CPU와 메모리 사이에 위치하여 **CPU가 이해하는 논리주소 <-> 메모리가 이해하는 물리 주소**로 변환하는 역할을 한다.

## 스와핑
- 입출력 작업으로 대기 상태 혹은 오랫동안 사용되지 않은 프로세스처럼 메모리에 적재된 프로세스 모두가 현재 실행되고 있는 것은 아니다.
- 이러한 프로세스를 임시로 보조기억장치의 일부인 **스왑 영역**이라는 곳으로 이동하고 
  해당 프로세스를 이동 시킨 빈 공간에 다른 프로세스를 적재하여 실행한다.

- **스왑 아웃**: 현재 실행되지 않는 프로세스가 메모리에서 스왑 영역으로 옮겨지는 것  
- **스왑 인**: 스왑 영역에 있는 프로세스가 다시 메모리로 옮겨오는 것  

스왑 아웃 되었다가 다시 스왑인 될 때는 이전의 물리 주소와 다른 주소에 적재될 수 있다.

---

## 연속 메모리 할당
- 메모리의 사용자 영역에서 프로세스들을 연속적인 메모리 공간에 할당하는 방식

### 스와핑과 연속 메모리 할당의 단점
- **외부 단편화**: 프로세스가 메모리에서 교체되면서 
  프로세스 외부에 다른 프로세스가 적재될 수 없는 크기의 메모리 공간이 계속해서 생겨나게 되어 메모리 낭비를 초래
- **메모리 크기**: 물리 메모리보다 큰 프로세스는 적재할 수 없음

---

## 가상 메모리
- 실행하고자 하는 프로그램의 일부만 적재하여 실제 메모리보다 더 큰 프로세스를 실행할 수 있도록 만드는 메모리 관리 기법
- 보조기억장치의 일부를 메모리처럼 사용하거나 
  프로세스의 일부만 메모리에 적재하여 실제 메모리의 크기보다 더 크게 보이게 하는 기술이다.
