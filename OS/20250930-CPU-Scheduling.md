# CPU 스케줄링

운영체제는 다양한 프로세스와 스레드에 CPU의 사용을 배분함으로써 CPU 자원을 관리한다.  
이 때 CPU가 자원을 배분하는 방법이 바로 **CPU 스케줄링**이다.

---

## 우선 순위
모든 프로세스는 CPU의 자원을 필요로 한다. 따라서 운영체제는 공정하고 합리적인 CPU 자원을 프로세스에 할당해야 한다.  
CPU를 할당받을 프로세스는 우선순위가 다르기 때문에 운영체제는 우선 순위를 판단하여 PCB에 명시하고,  
우선순위가 높은 프로세스에 더 많이, 더 빨리 CPU 자원을 할당한다.  
주로 CPU의 활용률을 높게 유지할 수 있도록 우선 순위를 설정한다.

- **입출력 집중 프로세스**: 입출력 작업이 많은 프로세스  
- **CPU 집중 프로세스**: 복잡한 수학 연산이나 그래픽 처리 작업을 담당하는 프로세스  

입출력 집중 프로세스는 실행보다는 입출력을 위한 대기 상태에 더 많은 시간을 소요한다.  
반대로 CPU 집중 프로세스는 실행 상태에 더 많이 머무르게 된다.  

👉 대기를 더 많이 해야 하는 입출력 집중 프로세스에 먼저 CPU를 할당해 작동시킨 뒤, CPU 집중 프로세스에 집중적으로 CPU를 할당한다.  
입출력 집중 프로세스는 대기 시간이 길기에 먼저 처리하면 CPU를 온전히 다른 프로세스에 집중시킬 수 있기 때문이다.

---

## 스케줄링 큐
CPU, 메모리 등 시스템의 자원은 한정되어 있기에 운영체제는 모든 프로세스에게 자원을 할당할 수 없다.  
운영체제는 자원을 이용하고 싶다면 줄을 서도록 하는데, 이것이 바로 **스케줄링 큐**이다.  

특정 자원을 사용하고 싶은 프로세스의 PCB를 해당 자원 큐에 삽입한다.  

운영체제가 관리하는 큐에는 다양한 종류가 있지만 주로 다음과 같다:
- **대기 큐**: 입출력 장치 사용 등으로 대기 중인 프로세스의 PCB가 적재됨
- **준비 큐**: 준비 상태인 프로세스의 PCB가 삽입됨  

대기 상태 혹은 준비 상태로 전환 시 PCB가 큐의 마지막에 삽입된다.

---

## 선점형 스케줄링
어떤 프로세스가 CPU를 할당받아 사용하는 중임에도 운영체제가 해당 프로세스의 CPU 자원을 해제하고 다른 프로세스에게 할당할 수 있는 스케줄링

### 장점
- 더 급한 프로세스가 끼어들어 CPU를 사용할 수 있기에 CPU 독점 방지  
- 여러 프로세스에 골고루 CPU 자원 배분 가능  

### 단점
- 문맥 교환 과정에서의 오버헤드 발생 가능  

---

## 비선점형 스케줄링
어떤 프로세스가 CPU를 사용하고 있다면 그 프로세스가 종료되거나 스스로 대기 상태에 접어들기 전에는 다른 프로세스가 끼어들 수 없는 스케줄링

### 장점
- 선점형 스케줄링에 비해 오버헤드 빈도가 적음  

### 단점
- 어떤 프로세스가 CPU 자원이 필요해도 CPU를 사용 중인 프로세스가 끝나야만 사용 가능  

---

## 스케줄링 알고리즘 종류

1. **선입 선처리 스케줄링 (FCFS, First Come, First Served)**  
   준비 큐에 삽입된 순서대로 CPU를 할당하는 방식이다.  
   먼저 삽입된 프로세스의 실행 시간이 길면 후에 삽입된 프로세스의 실행이 지연되는 **호위 효과**가 발생할 수 있다.

2. **최단 작업 우선 스케줄링 (SJF, Shortest Job First)**  
   준비 큐에 삽입된 프로세스 중 CPU 이용 시간이 가장 짧은 프로세스 먼저 실행하는 스케줄링 방식

3. **라운드 로빈 스케줄링**  
   선입 선처리 방식에 타임 슬라이스가 혼용된 형태의 스케줄링  
   준비 큐에 삽입된 순서대로 CPU를 할당하지만 정해진 시간만큼만 사용한 뒤, 다시 큐의 맨 뒤에 삽입된다.

4. **최소 잔여시간 우선 스케줄링 (SRT, Shortest Remaining Time)**  
   최단 작업 우선 스케줄링과 라운드 로빈을 결합한 방식  
   정해진 타임슬라이스만큼 사용하되, 남아 있는 작업 시간이 가장 적은 프로세스를 그 다음으로 실행한다.

5. **우선 순위 스케줄링**  
   프로세스에 우선순위를 부여하고 가장 높은 우선 순위를 가진 프로세스부터 실행하는 방식  
   - 단점: 우선 순위가 낮다면 큐에 먼저 삽입되었더라도 실행이 계속 연기되는 **아사(Starvation)** 현상 발생 가능  
   - 해결책: **에이징(Aging)** 기법 → 오랫동안 대기한 프로세스의 우선 순위를 점차 높여 아사 현상 방지

6. **다단계 큐 스케줄링**  
   우선 순위 방식에서 발전한 형태로, 우선순위 별로 여러 개의 준비 큐를 사용한다.  
   우선순위가 가장 높은 큐를 먼저 처리하고 해당 큐가 비어있으면 다음 우선순위 큐를 처리한다.  
   - 문제점: 우선순위가 높은 순으로 실행되어 아사 현상 발생 가능

7. **다단계 피드백 큐 스케줄링**  
   다단계 큐 방식과 달리 프로세스들이 **큐를 이동**한다.  
   큐에 새롭게 진입하는 프로세스는 우선순위가 가장 높은 큐에 삽입되고 타임 슬라이스만큼 실행.  
   이후 실행이 끝나지 않았다면 다음 우선순위 큐에 삽입된다.  
   - CPU 집중 프로세스: 점점 낮은 우선순위 큐로 이동  
   - 입출력 집중 프로세스: 높은 우선순위 유지  
   - 아사 현상 방지를 위해 **에이징 기법**을 적용할 수도 있다.
