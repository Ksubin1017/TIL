# 그래프

정점(Vertex)이라 불리는 데이터를 간선(Edge) 혹은 링크(Link)로 연결한 형태의 자료구조
기본적으로 데이터 간의 연결 관계를 표현하는 자료구조

---

## 그래프와 트리의 관계

- 트리 역시 그래프의 일종으로 사이클을 형성하지 않고 노드 간 상하 관계(부모-자식 관계)를 갖는다.  
- 반면 그래프는 사이클을 형성할 수 있고 이웃한 정점끼리 상하 관계를 갖지 않는다.

---

## 그래프의 종류

### 연결 그래프
- 그래프 상 임의의 두 정점 사이에 **경로가 존재하는 그래프**  
-> 임의의 정점들을 간선으로 이을 수 있다면 연결 그래프

### 비연결 그래프 
- 어떤 정점 사이에 경로가 **존재하지 않는 그래프**

### 방향 그래프 
- **간선에 방향이 있는 그래프**  
  → 예: A → B (A에서 B로만 이동 가능)

### 무방향 그래프
- **간선에 방향이 없는 그래프**  
  → 예: A — B (A에서 B로, B에서 A로 모두 이동 가능)

### 가중치 그래프 
- 간선에 가중치가 부여된 그래프  
- 가중치는 양수, 음수 모두 가능  
  → 예: 거리, 비용, 시간 등

### 서브 그래프
- 특정 그래프의 일부 정점과 간선으로 구성된 그래프

---

## 그래프의 구현과 저장 방식

### 인접 행렬 
- N × N 크기의 행렬로 그래프를 표현하는 방법  
- N은 정점의 개수를 의미하며 두 정점이 연결되어 있으면 1, 연결되지 않았으면 0으로 표시  
- 무방향 그래프의 경우 행렬의 대각선을 기준으로 대칭을 이룬다.
  → 예: 정점 N과 M이 연결되어 있다면  
    - `matrix[N][M] = 1`  
    - `matrix[M][N] = 1`

### 인접 리스트 
- 각 정점마다 연결된 정점들의 목록(연결 리스트)을 가지는 방식  
- 특정 정점에서 나가는 간선의 연결 대상 정점들을 리스트의 노드로 표현  

---

## 그래프 탐색

### 깊이 우선 탐색 (DFS, Depth-First Search)
- 더 이상 방문할 수 있는 정점이 없을 때까지 **최대한 깊이 탐색**하는 방식  

### 너비 우선 탐색 (BFS, Breadth-First Search)
- **인접한 모든 정점**을 우선적으로 방문한 뒤 방문한 정점들과 연결된 정점들을 차례로 방문하는 방식  

---

## 최단 경로 알고리즘

- 한 정점에서 목적지 정점까지 **가중치의 합이 최소가 되는 경로**를 구하는 알고리즘  
ex) **익스트라 알고리즘 (Dijkstra Algorithm)**
