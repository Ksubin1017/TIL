# 배열 (Array)
일정한 메모리 공간을 차지하는 여러 요소들이 순차적으로 나열된 자료구조
각 요소에는 **0부터 시작하는 인덱스**가 부여된다.

---

## 조회
- 인덱스로 요소를 식별하기 때문에 요소에 바로 접근이 가능  
- 요소 개수와 무관하게 일정한 시간 복잡도를 가짐  
  👉 **O(1)**  
- 단, 인덱스를 모르는 경우 원하는 데이터를 찾기 위해 모든 요소를 탐색해야 함  
  👉 **O(n)**

---

## 삽입 및 삭제
- 특정 요소를 추가하거나 삭제하기 위해서는 삽입/삭제 이후의 모든 요소들이 이동되어야 함
- 따라서 **재정렬이 필요**  
  👉 **O(n)**

---

## n차원 배열
- 기본적으로 한쪽 방향으로 요소가 나열되지만 **배열 속의 배열** 구조로 확장 가능  
- n개의 인덱스로 요소를 식별 가능

---

# 연결 리스트 (Linked List)
**노드(Node)**의 모음으로 구성된 자료구조  
연결 리스트의 모든 노드는 **메모리 내에서 순차적으로 저장될 필요가 없음**  
-> 불연속적인 메모리 공간을 사용

---

## 노드 (Node)
- 저장하고자 하는 데이터(data) 와 다음 노드의 위치(메모리 주소) 정보를 포함  
- 특정 노드에 접근하면, 그 노드가 가리키는 다음 노드로 접근 가능  

**헤드(Head)** : 연결 리스트의 첫 번째 노드  
**꼬리(Tail)** : 연결 리스트의 마지막 노드

---

## 조회
- 특정 요소에 접근할 때 **앞에서부터 순차 접근**  
  👉 **O(n)**

---

## 삽입 및 삭제
- 노드들이 연속적으로 저장되어 있지 않기 때문에 중간 삽입/삭제 시 **재정렬 불필요**  
  👉 **O(1)**

---

# 연결 리스트의 종류

## 싱글 연결 리스트 (Singly Linked List)
- 각 노드가 **다음 노드의 위치 정보만** 저장  
- 한쪽 방향으로만 연결되어 있어 **단방향 탐색만 가능**  
- 이전 노드로의 접근이 어려움

---

## 이중 연결 리스트 (Doubly Linked List)
- 각 노드가 이전 노드와 다음 노드의 위치 정보를 모두 포함  
- 양방향 탐색 가능  
- 탐색 성능은 향상되지만 위치 정보를 2개 저장하고 있기에 노드당 저장 공간이 더 많이 필요함

---

## 환형 연결 리스트 (Circular Linked List)
- **꼬리 노드가 헤드 노드를 가리키는 구조**  
- 노드들이 원형으로 연결되어 있음  
- 싱글, 이중 연결 리스트 모두로 구현 가능  
- 모든 노드를 여러 차례 순회해야 하는 경우 유용
